Heres an example of what we talked about today, lets just make this a long sentence so we can get a grasp of how the table will work

Consider the equation below that describes the triangular computation of the bubble sort:

X[i,j] = max(X[i-1, j], Y[i,j-1])

Y[i,j] = max(X[i-1, j], Y[i,j-1])

There are the usual special boundary conditions: at j=0, the bottom input is taken from the input array a_i, and on the diagonal boundary (i.e., where i=j+1) the “left” input is actually the Y value from the “southwest” point. The output of the whole computation is the X output of the rightmost column, plus the Y output of the top vertex [n-1,n-2].

Draw out the graph of this computation and then answer the following questions:

  • What is the length of the longest path in the graph, and draw at least three distinct such paths.
  • Consider a point/node [i,j] in the triangle. What is the length of the longest path in the graph that ends in [i,j].
  • A “schedule” is a function that assigns a “time-stamp” to each node. The bottom left corner of the triangle, [1,0] is a “source” of the graph, all its incoming edges are inputs. So it can be given a time stamp of t=0. Follow the reasoning and determine a schedule for any arbitrary node [i,j], in the graph. Draw a red “contour line” connecting all the points that are scheduled at time step 5, and a blue line through all those at time step 6.
  • [This one is hard] Can you prove or disprove the following statement: “Every node in the graph is on at least one longest path.”

Dimensional Analysis → GFLOPS/sec = flops/cycle x cycles/sec x cores x sockets we know: flops/cycle; frequency (cycles/sec); # cores; # sockets
